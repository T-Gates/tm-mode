# teammode — 팀의 기억을 켜는 스위치

> **Turn your team mode on.** 각자 쓰는 AI 코딩 에이전트에 "팀 모드"를 더하는 오픈소스 툴킷.
> 우리 팀(Acme, 3인)이 6주째 매일 실전에서 쓰고 있는 물건입니다.

---

## 1. 어떤 문제를 푸나

- "**그 결정, 회의 때 했는데 왜 그랬는지 기억하는 사람이 없다.**"
- 팀원들이 각자 AI 에이전트와 일하면서, 그 안에서 일어난 결정·삽질·발견이 **세션이 닫히면 휘발**된다.
- 누가 지금 뭘 하는지 알려면 결국 또 회의, 또 보고.

teammode는 에이전트 세션에서 일어나는 일을 **팀 공유 git 레포에 자동 누적**하고, 다음 세션을 열면 **팀 전체의 맥락이 자동 주입**되게 한다. "지금 팀 상황 어때?" 한 마디면 팀원 3명의 작업·블로커·일정이 좌르륵 나온다.

위키·Notion과 다른 점 한 줄: **사람이 쓰는 문서는 안 쓰게 되고, 쓴 문서는 안 읽게 된다.** 여기서는 쓰는 것(훅이 자동 누적)도 읽히는 것(세션 시작 시 자동 주입)도 자동이다.

## 2. 핵심 강점 5가지

| # | 강점 | 내용 |
|---|---|---|
| 1 | **도입이 쉽다** *(7월 공개판)* | 온보딩 = **설정 마법사**. 에이전트가 대화로 묻고 config를 직접 완성 — 손으로 편집할 설정 파일 0개. 사람 몫은 각 서비스 토큰 받아오는 심부름 **~10분**뿐 |
| 2 | **크로스에이전트** *(7월 공개판)* | 팀원마다 에이전트가 달라도(Claude Code / Codex / Hermes) **한 팀 메모리를 공유**. 혼성 팀 OK — 한 에이전트로 통일을 강요하지 않음 |
| 3 | **보고 비용 제로** | 세션로그가 훅으로 자동 누적 → **팀이 한 일과 진행 상황이 데이터로 남는다** — 보고하러 일을 멈출 필요가 없다. 주간 보고·스탠드업 준비가 "이미 쌓여 있는 걸 조회"로 바뀜 |
| 4 | **의사결정 추적** | 결정의 **근거와 접었던 대안**까지 기록이 표준 포맷. "그 결정 왜 했더라"라는 질문의 종말 |
| 5 | **도구는 그대로** *(슬롯 구조는 7월 공개판)* | issues/chat/docs/calendar **슬롯에 자기 도구를 꽂는 구조**. 공개판 기본 지원은 Linear/Slack/Notion/Google Calendar, Jira 등은 provider 팩으로 순차 확장. 빈 슬롯도 정상 상태(Slack만 있어도 시작 가능) |

## 3. 어떻게 동작하나 (1분 버전 — 7월 공개판 플로우)

```
리더:  팀 레포 생성(template) → 온보딩 마법사와 대화로 설정 (1회)
팀원:  git clone && ./install.sh <이름>          # 합류는 이 한 줄
일상:  에이전트에서 "teammode on" → 팀 맥락 주입 → 일하면 로그가 알아서 쌓임
```

- 데이터는 전부 **팀 소유 private git 레포** — 외부 서버 없음, 벤더 락인 없음, 마크다운이라 에이전트 없이도 읽힌다.
- 규율은 사람이 아니라 **훅이 든다**: 로그 안 쓰고 30분 지나면 에이전트가 알아서 기록하게 리마인드가 박힌다. 습관 형성을 의지에 안 맡김.

## 4. 정직한 기대치 — 우리 6주 사용기

| 시기 | 실제로 일어나는 일 |
|---|---|
| **1주차** | 솔직히 귀찮다. 로그 리마인드가 잔소리처럼 느껴짐. 가치 체감 거의 없음 |
| **2주차~** | 복리 시작. "이거 지난주에 왜 이렇게 했지?"가 검색 한 번으로 끝나기 시작 |
| **6주차 (현재)** | 시스템이 우리보다 팀을 잘 기억한다. 실사례: 멤버 A의 질문과 멤버 B가 따로 적어둔 플래그가 같은 지점을 가리키는 걸 에이전트가 잡아냄 |

- 우리는 이 툴킷의 **1호 고객**이다. teammode 프로젝트 운영 자체를 teammode로 한다 (독푸딩).
- 비용의 골짜기는 1주차 하나뿐이고, 그걸 훅 자동화로 메우는 게 이 설계의 전부다.

## 5. 현재 상태와 로드맵 (과장 없이)

| 구분 | 항목 |
|---|---|
| **지금 있는 것** | 팀 메모리 표준(스펙 문서화 완료) · 세션로그 자동 누적 훅 · 팀 컨텍스트 주입 · Linear/Slack/Notion/GCal 연동 · 회의록/결정사항/지식베이스 관리 스킬 — 전부 우리가 매일 사용 중 |
| **공개판 작업 중** (6월 말 구현 착수 → 7월 공개 목표) | 온보딩 설정 마법사 · 크로스에이전트 어댑터(Codex/Hermes) · 서비스 슬롯 일반화(Jira 등 provider 확장 구조) — 설계 스펙 초안 완료, 구현은 6월 말 착수 |
| **로드맵 (아직 없음)** | 팀 대시보드 · 충돌 레이더(두 사람이 같은 지점을 건드리면 감지) — 데이터(세션로그·summary)는 지금 포맷부터 쌓이게 설계됨 |

## 6. 도입한다면

- **오늘 바로 볼 수 있는 것**: 우리 팀 레포에서 "지금 팀 상황" 라이브 데모 — 6주치 실데이터로 시연.
- **파일럿 제안**: 7월 공개 목표 기준, 팀원 2~3명으로 2주 — 1주차 골짜기를 넘기는 게 판단 기준이라 최소 2주.
- 리더가 할 일: 팀 레포 만들기 + 마법사와 설정 대화 + 토큰 심부름 ~10분. 팀원: clone 한 줄.
- 안 맞으면 버리는 비용: 레포 삭제 끝. 데이터는 마크다운이라 어차피 팀 소유.
- 먼저 도입하는 팀의 실전 피드백이 그대로 스펙에 반영된다 — 지금이 요구사항을 꽂기 가장 좋은 시점.
